## 3. 功能测试用例

### 3.1 持仓列表显示 (TC-001)

**测试目的**: 验证持仓列表正确显示

**前置条件**: 用户已登录，有初始持仓数据

**测试步骤**:
1. 打开 http://43.160.193.165/minirock-v2.html
2. 等待页面加载完成

**预期结果**:
- [ ] 页面顶部显示"灵犀投资助手"标题
- [ ] 健康度圆环显示分数（如78分）
- [ ] "我的持仓"区域显示3只股票
- [ ] 每只股票显示：名称、代码、持股数量、当前价格、涨跌幅、健康度
- [ ] 组合总价值、总盈亏正确计算

**验收标准**:
- 持仓数据加载时间 < 3秒
- 价格显示为人民币格式（¥1,234.56）
- 涨跌幅为正显示红色，为负显示绿色

---

### 3.2 添加持仓 (TC-002)

**测试目的**: 验证可以正常添加股票到持仓

**测试步骤**:
1. 在搜索框输入"平安银行"
2. 点击搜索结果中的"平安银行"
3. 在弹窗中输入：持股数量=500，持仓成本=10.5
4. 点击"确认添加"

**预期结果**:
- [ ] 搜索能正确显示股票名称和代码
- [ ] 弹窗正确显示股票名称和代码
- [ ] 可以输入持股数量（正整数）
- [ ] 可以输入持仓成本（小数）
- [ ] 添加成功后显示Toast提示
- [ ] 持仓列表自动刷新，显示新增股票
- [ ] 组合总价值重新计算

**验收标准**:
- 添加操作响应时间 < 2秒
- 添加后数据持久化（刷新页面后仍在）
- 重复添加同一股票应提示"该股票已在持仓中"

---

### 3.3 删除持仓 (TC-003)

**测试目的**: 验证可以正常删除持仓

**测试步骤**:
1. 在持仓列表中找到一只股票
2. 点击该股票右侧的🗑️删除按钮
3. 在确认对话框中点击"确定"

**预期结果**:
- [ ] 每只股票右侧显示删除按钮
- [ ] 点击删除按钮弹出确认对话框
- [ ] 确认后显示"删除成功"提示
- [ ] 持仓列表自动刷新，该股票消失
- [ ] 组合总价值重新计算

**验收标准**:
- 删除操作响应时间 < 1秒
- 删除后数据持久化（刷新页面后不会恢复）
- 取消删除时数据保持不变

## 4. 接口测试用例

### 4.1 API连通性测试

```bash
# 测试持仓列表API
curl http://43.160.193.165/api/portfolio/holdings?user_id=demo_user

# 预期返回：JSON格式持仓数据
```

### 4.2 添加持仓API

```bash
curl -X POST "http://43.160.193.165/api/portfolio/holdings?user_id=demo_user" \
  -H "Content-Type: application/json" \
  -d '{"symbol":"000001","name":"平安银行","quantity":100,"avg_cost":10.5}'

# 预期返回：{"success":true,"message":"已添加 平安银行 到持仓"}
```

### 4.3 删除持仓API

```bash
curl -X DELETE "http://43.160.193.165/api/portfolio/holdings/000001?user_id=demo_user"

# 预期返回：{"success":true,"message":"已删除股票 000001"}
```
