## 📝 重要规则更新（2026-03-13）

### 交付模式变更

**之前**: 按截止时间交付  
**现在**: **完成即交付，做完立即更新状态**

```diff
- ❌ 不需要写截止时间
+ ✅ 做完立即标记"已完成"
+ ✅ 有问题立即汇报阻塞
+ ✅ 实时更新进度（每30分钟检查一次）
```

### 任务状态流转

```
📋 待领取 → 🔄 进行中 → ✅ 已完成
              ↓
            🔴 阻塞中（有问题立即汇报）
```

## 🔄 Agent 工作流程

### Step 1: 查看任务

```bash
ssh ubuntu@43.160.193.165
cat /var/www/familystock/tasks/$(date +%Y-%m-%d).md
```

### Step 2: 领取任务

找到任务后，立即更新状态：

```markdown
**状态**: 🔄 进行中
**领取时间**: 2026-03-13 10:30
**负责Agent**: @方舟
```

### Step 3: 执行并实时更新

**每30分钟检查一次任务文件**，边执行边更新：

```markdown
#### 当前进度
**完成状态**: 60%
**已完成功能**: 
- [x] 设计稿初稿
- [ ] 设计评审
- [ ] 设计文档输出

#### 问题与支持
**遇到的问题**: 需要确认颜色规范
**需要的支持**: 请先生确认品牌色值
```

### Step 4: 完成即交付

**完成后立即更新**（不要等截止时间）：

```markdown
**状态**: ✅ 已完成
**完成时间**: 2026-03-13 15:00
**实际耗时**: 4小时

#### 完成记录
**完成状态**: 100% 完成
**解决方案**: 简述如何完成的
**遇到的问题**: 如有问题说明原因
**需要的支持**: 如需协助说明需求
**交付物**: 链接/文件位置
```

---

### Step 5: 测试验收流程

**完成后进入测试环节：**

```
Agent完成任务 → 生成测试任务 → 双人测试 → 验收通过 → 进行下一个任务
```

#### 5.1 生成测试任务
**项目经理（小七）执行：**
- 为完成的任务创建测试任务
- 任务ID: `TEST-{原任务ID}`

#### 5.2 双人测试机制

| 测试人员 | 职责 | 测试重点 |
|----------|------|----------|
| **方舟** | UX/UI测试 | 界面设计、交互体验、视觉效果 |
| **小七** | 功能测试 | API接口、功能逻辑、边界情况 |

#### 5.3 测试任务模板
```markdown
### TEST-YYYYMMDD-XXX [📋 待领取]
| 字段 | 内容 |
|------|------|
| **任务ID** | TEST-YYYYMMDD-XXX |
| **任务标题** | 测试：XXX功能验证 |
| **测试对象** | TASK-YYYYMMDD-XXX |
| **测试人员** | @方舟 + @小七 |
| **状态** | 🧪 测试中 |

#### 测试项
- [ ] 方舟：UI/UX测试通过
- [ ] 小七：功能测试通过

#### 验收结果
- [ ] 🟢 全部通过 → 进行下一个任务
- [ ] 🔴 有问题 → 返回修复
```

#### 5.4 验收标准
- **🟢 绿灯**: 双方测试都通过，Agent可进行下一个任务
- **🔴 红灯**: 任一方测试不通过，返回Agent修复

---

### Step 6: 遇到阻塞立即汇报

如遇到问题，**立即标记**：

```markdown
**状态**: 🔴 阻塞中
**阻塞时间**: 2026-03-13 14:00

#### 问题与支持
**遇到的问题**: API接口返回500错误
**需要的支持**: 需要灵犀协助排查后端问题
**阻塞原因**: 等待后端接口修复
```

## ⚡ 优先级定义

| 优先级 | 含义 | 响应时间 | 示例 |
|--------|------|----------|------|
| **P0-紧急** | 阻塞性问题，影响整体进度 | 立即处理 | 系统崩溃、核心功能无法使用 |
| **P1-高** | 重要功能 | 尽快完成 | 关键页面开发、重要Bug修复 |
| **P2-中** | 常规任务 | 按序完成 | 功能优化、文档编写 |
| **P3-低** | 改进项 | 有时间处理 | 代码重构、性能优化 |

## 📞 沟通规则

1. **任务完成**：立即更新任务文件状态为"已完成"
2. **遇到阻塞**：立即标记"🔴 阻塞中"并在飞书汇报

## ✅ 关键原则

### 完成即交付原则
- 不要等截止时间
- 做完立即标记"✅ 已完成"
- 让项目经理立即知道

### 阻塞立即汇报原则
- 遇到问题不要硬撑
- 立即标记"🔴 阻塞中"
- 在飞书群说明问题

### 实时更新原则
- 每30分钟检查任务文件
- 进度变化立即记录
- 保持信息透明
