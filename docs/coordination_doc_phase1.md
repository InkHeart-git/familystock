#### 数据来源
- **Tushare Pro API**: A股日K、指数数据
- **同步频率**: 每日收盘后18:00
- **数据量预估**: 
  - 日K: ~5,000股票 × 250交易日 = 125万条/年
  - 指数: 8个指数 × 1条/天

#### SQL验证脚本
```sql
-- 检查日K数据量
SELECT COUNT(*) FROM stock_daily;
-- 预期: > 100万条

-- 检查指数数据
SELECT ts_code, name, close, pct_chg FROM index_quotes;
-- 预期: 8条记录，close > 0

-- 检查最新日期
SELECT MAX(trade_date) FROM stock_daily;
-- 预期: 20260319 或最新交易日
```

#### 验收标准
- [ ] `stock_daily` 表 > 100万条记录
- [ ] `index_quotes` 表 close 字段有有效数值
- [ ] 上证指数显示正常（约3000-4000点）
- [ ] 个股K线API返回历史数据

## 三、协作边界

### ✅ 可以并行（无冲突）

```
┌─────────────────┐     ┌─────────────────┐
│   玲珑（前端）   │     │   灵犀（数据）   │
│  ├─ 改品牌名    │     │  ├─ 同步日K     │
│  ├─ 删冗余页面  │     │  ├─ 同步指数    │
│  └─ 统一样式    │     │  └─ 更新行情    │
└─────────────────┘     └─────────────────┘
         ↓                       ↓
    互不影响               互不影响
```

### ⚠️ 需要协调（避免冲突）

| 场景 | 协调方式 |
|------|---------|
| 同时修改同一文件 | 提前沟通，分时段修改 |
| 数据库结构调整 | 灵犀主导，玲珑配合验证 |
| API接口变更 | 后端先行，前端适配 |
| 版本发布 | 统一时间，一起发布 |
