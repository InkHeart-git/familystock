## 🔄 跨Agent协作流程

### 场景1: 前端需要新API

1. **玲珑** 创建任务: "需要获取用户持仓历史API"
   - 分配给: @灵犀
   - 优先级: P1-高
   - 包含: 接口需求文档

2. **灵犀** 领取并开发API

3. **灵犀** 完成后，任务状态更新为 ✅ 已完成
   - 包含: API文档链接

4. **玲珑** 看到完成后，领取前端开发任务

### 场景2: UI设计需要技术可行性评估

1. **方舟** 创建任务: "确认AI分析卡片动效可行性"
   - 分配给: @灵犀
   - 优先级: P2-中

2. **灵犀** 评估后回复可行性结论

3. **方舟** 根据结论继续设计

### 场景3: 发现Bug需要修复

1. **方舟** (测试) 创建任务: "首页搜索框在iOS上无法聚焦"
   - 分配给: @玲珑 (前端)
   - 优先级: P1-高
   - 包含: Bug描述、复现步骤、截图

2. **玲珑** 修复后更新任务

3. **方舟** 验证后关闭任务

## 🆘 遇到问题怎么办？

| 问题类型 | 处理方式 |
|----------|----------|
| 任务不清晰 | 在任务文件中添加问题说明，@项目经理 |
| 技术难点 | 标记为 🔴 阻塞，说明需要的支持 |
| 时间不够 | 标记为 ⏸️ 暂停，说明原因和预计完成时间 |
| 需要其他Agent协助 | 创建新任务分配给对应Agent，标记依赖关系 |
| 紧急情况 | 飞书直接联系 @先生 或 @小七 |

## 💡 最佳实践

1. **及时更新** - 不要等到完成才更新，进度随时同步
2. **详细记录** - 问题描述越详细，项目经理越容易协助
3. **主动沟通** - 遇到阻塞不要等待，立即标记
4. **预留缓冲** - 预估工时增加20%缓冲时间
5. **文档同步** - 代码/设计修改后，同步更新相关文档
